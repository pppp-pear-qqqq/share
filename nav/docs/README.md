# 定期ゲーム汎用ナビゲーターシステム
ナビゲーターとは、ロキサ(sssloxisa)氏作のオリジナルキャラクター登録型ゲーム『シャリシャリ』に搭載されていたシステムです。
他のキャラクターが設定したナビゲーターとしてのセリフをUI上に適宜表示していくことで嬉しくなるという素晴らしいもの。

これはそのナビゲーターシステムを汎用化し、あらゆる定期ゲームに拡張機能として実装できるようにするためのベースシステムです。

# 使用方法
Tampermonkey用のユーザースクリプトを作成し、そこから`navigator.min.js`をインポートします。
同時にアプリごと個別に用意するスタイルシートも読み込み反映します。
ユーザースクリプトの作例は`app/google.user.js`を参照。

# DB APIについて
`UserNavigator`はコンストラクタ引数に`db_api`を受け取ります。
これはナビゲーター設定のデータベースとして使用するAPIのURLを指定します。
基本的にはGoogleスプレッドシートにGAS(AppScript)を適用してAPI化して運用することになると思いますが、自分で同じ仕様のAPIを作成しても構いません。

アプリ・ゲームごとにこのDBは変更する予定です。（全て共通にすると規模感次第では重たくなりうるため）
なので自分で他システム対応版を作りたい場合は自分でDB APIを用意してください。
`other/gas.js`をそのままAppScriptとして適用すればいいと思います。また、スプレッドシートの行列は不要分は削除しておいた方がパフォーマンスが良くなります。

## API仕様
細かいことは`other/gas.js`を見てもらうのが一番確実だと思います。

共通の仕様として、リクエストの成功／失敗はレスポンスコードではなくContent-Typeやボディによって識別します。
（GASの制約による仕様。自分でAPI作るなら変えてもいいとは思いますが、レスポンスコード分岐を前提にベースシステムが作られていることには留意してください。）
### GET
ナビゲーターデータを受け取ります。
クエリ文字列の形式でパラメータ`key`を受け取り、`key`と完全一致するレコードを探してその値となるJSON文字列を返します。
成功した場合は`"Content-Type":"application/json"`を返し、失敗なら`"Content-Type":"text/plain"`を返します。
### POST
ナビゲーターデータを設定します。
FormDataまたはJSONでパラメータを渡し（どちらでもOK）、`key`が一致するものがあれば更新、なければ新規作成を行います。
成功でも失敗でも`"Content-Type":"text/plain"`を返しますが、成功の場合はボディとして`"Updated"`または`"Created"`を返します。

## 実装予定
- トリガーを増やす
- さまざまなゲーム用のTampermonkeyスクリプト作成
## できたら実装したい
- カスタムトリガー、およびカスタムトリガーオプション
	- サイトごとの固有トリガーなどのため
- (実装じゃないけど)ナビゲーターシステムのプロジェクト切り出し
	- 現状`share`という私がユーザースクリプトや簡単なwebページ等公開するエリアに纏めて置いているのですが、思ったより規模感が大きくちゃんとしたプロジェクトになりそうだったので
	- まあでも面倒だしやらないかな　どうせ私以外にメンテナなんかいないだろうし
## 実装しないもの
- 日付トリガーオプション(特定の日になったら表示される)
	- 用例としては誕生日や記念日のお祝いなどがあるかと思いますが、それはナビゲーターじゃなくて自分で言った方がいいと思うので実装しません
	- とはいえカスタムトリガー作れるようになったらこれも実装は簡単だと思います
